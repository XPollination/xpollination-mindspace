## MANAGEMENT ABSTRACT

**What:** A visual project management dashboard that enables real-time alignment between the PDCA Agent (Claude) and the Human (Thomas) during software development.

**Why:**
- Reading 800-line PDCA documents is inefficient for status tracking
- Human needs visual, interactive way to provide feedback on priorities
- Drag-and-drop reordering communicates intent faster than text
- Hierarchical view (phases → items → sub-items → requirements) matches mental model
- Must exist BEFORE we can effectively build the Agentic Development Framework

**Outcome:** A Hugo-based web application at `https://10.33.33.1/projectmanagement/` where:
1. PDCA Agent writes structured YAML files
2. Hugo renders them as interactive dashboard
3. Human views status, dependencies, progress
4. Human drags/drops to reorder priorities
5. Changes sync back to inform PDCA Agent

**Analogy:** Like Trello/Jira, but:
- Data stored as YAML (AI-readable/writable)
- Rendered statically via Hugo (no backend required)
- Designed for human-AI collaboration, not human-human

#### 1.2 Technology Stack Decision

| Layer | Choice | Rationale |
|-------|--------|-----------|
| **Static Site Generator** | Hugo | Already planned, fast, Go templates |
| **Reactivity** | Alpine.js | Lightweight (15kb), declarative, no build step |
| **Drag-and-Drop** | SortableJS | Mature, touch support, nested lists |
| **Styling** | Tailwind CSS | Utility-first, rapid prototyping |
| **Data Format** | YAML | Human-readable, AI-writable, Hugo-native |
| **Dependency Graph** | Mermaid.js | Already in Hugo ecosystem |
| **Icons** | Heroicons | SVG, works with Tailwind |

#### 1.3 Why NOT Other Options

| Option | Rejected Because |
|--------|------------------|
| React/Vue SPA | Overkill, requires build pipeline |
| Vanilla JS only | Too much boilerplate for reactivity |
| Server-side app | Unnecessary complexity, Hugo sufficient |
| Database backend | YAML files sufficient, simpler deployment |
