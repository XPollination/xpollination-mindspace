## MANAGEMENT ABSTRACT

**What:** Implement the PM tool features defined by the Recursive PDSA Framework. Transform the current flat dashboard into a tree-aware, scope-stack-navigable tool that visualizes hierarchical PDSA cycles and their relationships.

**Why:**
- Current dashboard shows a flat list of phases/items — no visibility into PDSA depth
- No way to see "where are we in the tree" (scope stack)
- No way to navigate between parent and child PDSAs
- No way to see scope breaks or quality gate status
- Without these capabilities, human-agent alignment degrades at every depth transition

**Outcome:** A dashboard where:
1. The scope stack is always visible (breadcrumb: Root > PM Tool > Current)
2. PDSA tree structure is navigable (drill down into children, back up to parents)
3. Scope breaks are visually distinct from regular items
4. Quality gate status is visible per item
5. PDSA phase (Plan/Do/Study/Act/Complete) is shown per node

### 2. Target State (v2 — What We Need)

#### 2.1 Scope Stack Breadcrumb

A persistent breadcrumb bar showing the current position in the PDSA tree:

```
┌──────────────────────────────────────────────────────────────────────┐
│ 📍 Agentic Framework (paused) › PM Tool (active) › v2 Impl (active) │
└──────────────────────────────────────────────────────────────────────┘
```

- Each node is clickable → navigates to that PDSA's context
- Current scope is highlighted
- Paused/blocked parents show their status
- Always visible at the top of the dashboard

**Data source:** `meta.scope_stack` array in context.yaml

#### 2.2 PDSA Phase Indicator

Each phase/item shows which PDSA phase it is in:

```
┌─────────────────────────────────────────────────────────────┐
│ 🔵 Infrastructure Setup                          [COMPLETE] │
│ ● Plan  ● Do  ● Study  ● Act  ✅                           │
├─────────────────────────────────────────────────────────────┤
│ 🔵 Recursive PDSA Framework                      [COMPLETE] │
│ ● Plan  ● Do  ● Study  ● Act  ✅                           │
├─────────────────────────────────────────────────────────────┤
│ 🔵 PM Tool v2 Implementation                   [PLAN PHASE] │
│ ◉ Plan  ○ Do  ○ Study  ○ Act                               │
└─────────────────────────────────────────────────────────────┘
```

**Data source:** `pdsa_phase` field per item (plan/do/study/act/complete)

#### 2.3 Scope Break Visual Indicator

When a scope break occurs (child PDSA spawned), show it distinctly:

```
┌─────────────────────────────────────────────────────────────┐
│ ⚡ SCOPE BREAK: Recursive PDSA Framework                     │
│ Reason: "Need to understand hierarchical PDCA flow first"    │
│ Parent paused: PM Tool Phase B                               │
│ Status: ✅ Complete → Returning to parent                    │
└─────────────────────────────────────────────────────────────┘
```

Scope breaks are styled differently (accent border, lightning bolt icon) to distinguish from regular items.

**Data source:** Phase with `type: "scope_break"` in context.yaml

#### 2.4 Quality Gate Display

Per item, show the quality gates and their pass/fail status:

```
┌─────────────────────────────────────────────────────────────┐
│ Quality Gates:                                               │
│ ✅ QG-RF-1: Terminology unambiguous                         │
│ ✅ QG-RF-2: Tree model matches workflow                     │
│ ✅ QG-RF-3: PM tool requirements implementable              │
│ ⬜ QG-RF-4: Data model backward-compatible                  │
└─────────────────────────────────────────────────────────────┘
```

**Data source:** `quality_gates` array per item in context.yaml

#### 2.5 PDSA Tree Navigation

Allow drilling into child PDSAs and navigating back:

- Items that have child PDSAs show a "drill down" icon
- Clicking loads the child PDSA's context.yaml
- Breadcrumb updates to show new position
- "Back to parent" button to return up the tree

**Data source:** `children` array in pdsa_tree or separate context.yaml files per PDSA

#### 3.2 Data Model Changes

Extend currentContext.yaml:

```yaml
# New fields in meta:
meta:
  scope_stack:
    - id: "root"
      title: "Agentic Development Framework"
      status: "paused"
    - id: "pm-tool"
      title: "Project Management Tool"
      status: "active"

# New fields per phase:
phases:
  - id: "phase-scope-break"
    type: "scope_break"          # NEW: "phase" | "scope_break"
    scope_break_reason: "..."    # NEW: why this break occurred
    pdsa_phase: "complete"       # NEW: plan | do | study | act | complete

# New fields per item:
items:
  - id: "sb.1"
    pdsa_phase: "complete"       # NEW
    quality_gates:               # NEW
      - name: "Gate name"
        status: "pass"           # pass | fail | pending
```
